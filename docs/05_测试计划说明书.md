# 测试计划说明书

> **项目名称：** 综合物流管理系统
> **文档版本：** V1.0
> **编写日期：** 2026-04-18

---

## 1. 测试策略

### 1.1 测试分层

```
┌─────────────────────────────────────┐
│       E2E 测试（Playwright）          │ ← 最高层，完整用户流程
├─────────────────────────────────────┤
│       API 测试（pytest）             │ ← REST 接口覆盖
├─────────────────────────────────────┤
│       单元测试（JUnit + Mockito）      │ ← 核心业务逻辑
└─────────────────────────────────────┘
```

### 1.2 单元测试（JUnit 5）

| 模块 | 测试覆盖目标 |
|------|------------|
| WarehouseService | 入库/出库/库存增减逻辑 |
| OrderService | 订单创建/状态流转/费用计算 |
| TransportService | 运单分配/配送状态更新 |
| AuthService | JWT 颁发/验证/Refresh |

### 1.3 API 测试（pytest）

| 文件 | 覆盖场景 |
|------|---------|
| `test_auth.py` | 登录/登出/token刷新 |
| `test_warehouse.py` | 库位CRUD/入库/出库/库存查询 |
| `test_order.py` | 订单CRUD/状态流转 |
| `test_transport.py` | 运单/司机/车辆/跟踪 |
| `test_statistics.py` | 仪表盘数据 |

### 1.4 E2E 测试（Playwright）

| 文件 | 覆盖场景 |
|------|---------|
| `e2e/login.spec.ts` | 登录成功/失败/路由守卫 |
| `e2e/warehouse.spec.ts` | 库位列表/入库流程 |
| `e2e/order.spec.ts` | 创建订单/查看详情 |
| `e2e/transport.spec.ts` | 运单分配/状态更新 |

---

## 2. 测试环境

### 2.1 资源约束

- PostgreSQL/Redis/MinIO/RocketMQ 均使用 **Docker 容器**
- 后端 JAR 宿主机运行，限制堆内存 768MB
- 所有容器总内存 ≤ 2.5GB

### 2.2 测试数据库

测试使用独立的 PostgreSQL 数据库（与开发数据库分离），通过 `application-test.yml` 配置。

| 级别 | 通过率要求 |
|------|----------|
| 单元测试 | 核心业务方法覆盖率 ≥ 80% |
| API 测试 | 所有接口 200 响应正常 |
