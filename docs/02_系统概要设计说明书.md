## 3. 核心业务流

### 3.1 入库流程

```
客户下单（外部）
     ↓
订单创建（logistics-order）
     ↓ RocketMQ 消息
入库通知（logistics-warehouse）
     ↓
仓库操作员录入入库单
（选择库位，确认货品、数量）
     ↓
库存增加（logistics-warehouse 库存表）
     ↓
订单状态更新 → IN_WAREHOUSE
```

### 3.2 出库配送流程

```
客户下单
     ↓
订单创建
     ↓
仓库拣货出库
     ↓
运单创建（绑定司机+车辆）
     ↓ RocketMQ 消息
司机收到配送任务通知
     ↓
提货 → 在途中 → 送达
     ↓
订单状态更新 → DELIVERED
     ↓ RocketMQ 消息
客户收到送达通知
```

### 3.3 异步消息流（RocketMQ）

| 消息主题 | 生产者 | 消费者 | 用途 |
|---------|-------|-------|------|
| order-created | logistics-order | logistics-warehouse | 新订单触发入库备货 |
| order-shipped | logistics-order | logistics-transport | 出库触发配送任务 |
| order-delivered | logistics-transport | logistics-order | 配送完成更新订单状态 |
| inventory-low | logistics-warehouse | logistics-transport | 库存不足通知调度 |

## 6. 安全性设计

### 6.1 认证流程

```
用户登录 → POST /api/auth/login
     ↓
验证用户名密码（BCrypt）
     ↓
生成 JWT Access Token（2小时）
     ↓
返回 token 给前端
     ↓
前端存储在 localStorage
     ↓
后续请求 Header 携带：Authorization: Bearer <token>
```

### 6.2 权限控制

- **接口级**：Spring Security + Method Security
- **数据级**：用户绑定部门/仓库，查询时自动过滤
- **菜单级**：前端路由守卫 + 后端菜单权限码

    memory: 512m

    memory: 768m
    environment:
      - JAVA_OPTS=-Xms768m -Xmx768m

  backend:
    # 宿主机 JAR，不放容器

所有 Docker 容器内存总和 ≤ 2.5GB，剩余资源给宿主机、OpenClaw 和前端开发服务器。
