## ⚠️ 虚拟机资源约束（4核8G）

> **必须遵守。调试前先检查并释放资源。**

### 内存分配上限

| 组件 | 最大内存 | JVM 参数 |
|------|---------|---------|
| PostgreSQL 16 | 512MB | `--memory=512m` |
| Redis 7 | 256MB | `--memory=256m` |
| MinIO | 300MB | `--memory=300m` |
| RocketMQ NameServer | 200MB | `-Xms200m -Xmx200m` |
| RocketMQ Broker | **768MB** | **`-Xms768m -Xmx768m`（禁止默认值 2G）** |
| Spring Boot Backend | 768MB | `-Xms768m -Xmx768m -Xss512k` |
| 前端 Vite dev | 500MB | — |

**总上限：约 3.5GB，剩余 ~2.5GB 给宿主机 + OpenClaw**

### 调试前检查清单

```
调试项目前必做：
1. 检查残留进程：ps aux | grep java / grep node / grep mvn
2. 杀掉不需要的进程：kill <PID>
3. 确认 Docker 容器内存限制已设置
4. 不要同时运行多个 Java 进程
5. RocketMQ Broker 必须用 -Xms768m -Xmx768m 启动
```

### 常用清理命令

```bash
# 杀掉所有 Java 残留进程（谨慎使用）
ps aux | grep java | grep -v grep | awk '{print $2}' | xargs kill 2>/dev/null

# 杀掉所有 Node 残留进程
ps aux | grep node | grep -v grep | awk '{print $2}' | xargs kill 2>/dev/null

# 杀掉 Maven 残留进程
ps aux | grep maven | grep -v grep | awk '{print $2}' | xargs kill 2>/dev/null

# 查看内存使用
free -h
```
