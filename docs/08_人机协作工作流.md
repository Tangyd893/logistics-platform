## 1. 项目概述

### 1.1 项目目标

搭建一套面向中小型物流企业的综合管理平台，覆盖仓储、订单、运输三大核心模块，采用 Spring Boot 3 + React 18 轻量化技术栈，支持 Docker Compose 部署。

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Spring Boot 3 + Java 21 + PostgreSQL 16 + Redis 7 |
| 消息队列 | RocketMQ 5.x |
| 对象存储 | MinIO |
| 前端 | React 18 + Vite + TypeScript + Zustand |
| UI | shadcn/ui + Tailwind CSS |
| 图表 | ECharts 5 + Leaflet |
| API文档 | SpringDoc OpenAPI 3 |

## 5. ⚠️ 虚拟机资源管理规范

> **4核8G环境，必须遵守。每次调试前执行检查。**

### 5.1 调试前检查

```bash
# 检查残留进程
ps aux | grep -E 'java|node|mvn|gradle' | grep -v grep

# 查看内存余量
free -h

# 杀掉不需要的进程
kill <PID>
```

### 5.2 RocketMQ Broker 内存限制

**禁止使用默认值（2GB）**，必须手动设置：

```bash
# 正确启动方式
export NAMESRV_OPTS="-Xms200m -Xmx200m"
export BROKER_JVM_OPTS="-Xms768m -Xmx768m"
rocketmq namesrv &
rocketmq broker -n localhost:9876 &
```

### 5.3 启动顺序

```bash
# 1. Docker 基础设施
cd docker && docker compose up -d postgres redis minio rocketmq

# 2. 后端（先确认无残留进程）
java -Xms768m -Xmx768m -Xss512k -jar backend/target/logistics-backend-1.0.0.jar &

# 3. 前端
cd frontend && VITE_API_BASE_URL=http://localhost:8080 npx vite --port 3000
```

### 5.4 进程清理命令

```bash
# 一键清理所有残留进程（谨慎）
ps aux | grep -E 'java|node|mvn' | grep -v grep | awk '{print $2}' | xargs kill 2>/dev/null
```

## 6. 故障排查索引

### 6.1 后端起不来

| 现象 | 排查 |
|------|------|
| 端口占用 | `kill $(lsof -ti:8080)` |
| 数据库连不上 | 检查 Docker postgres 容器是否运行 |
| JWT 报错 | 检查 `application.yml` 中的 jwt secret |

### 6.2 前端起不来

| 现象 | 排查 |
|------|------|
| 端口占用 | `kill $(lsof -ti:3000)` |
| 依赖安装失败 | `rm -rf node_modules && npm install` |
| Vite 构建报错 | 检查 TypeScript 错误 |

### 6.3 RocketMQ 起不来

| 现象 | 排查 |
|------|------|
| 内存不足 | Broker 必须用 `-Xms768m -Xmx768m` |
| NameServer 连不上 | 检查 9876 端口 |
| Broker 连不上 | 检查 8080 端口 |
