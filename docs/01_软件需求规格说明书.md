> **项目名称：** 综合物流管理系统
> **文档版本：** V1.0

## 1. 引言

### 1.1 目的

本文档对综合物流管理系统的功能需求、非功能需求、用户特征、约束条件和假设依赖进行完整描述，作为后续系统设计、开发和测试的依据。

### 1.2 范围

本系统为面向中小型物流企业的综合管理平台，核心功能包括：

- 仓储管理（库位、库存、入库、出库）
- 订单管理（订单创建、状态流转、查询）
- 运输管理（运单、司机、配送跟踪）
- 系统管理（用户、角色、权限）

### 1.3 定义、缩略语

| 术语/缩写 | 定义 |
|-----------|------|
| SKU | Stock Keeping Unit，库存量单位 |
| WMS | Warehouse Management System，仓储管理系统 |
| TMS | Transportation Management System，运输管理系统 |
| OMS | Order Management System，订单管理系统 |
| JWT | JSON Web Token，基于令牌的身份认证机制 |
| SPU | Standard Product Unit，标准产品单位 |

## 2. 总体描述

### 2.1 产品背景

国内中小型物流企业普遍面临仓库管理粗放、运输调度效率低、订单状态不透明等问题。本系统通过数字化仓储和运输全流程，实现库存实时可视、订单全程可追溯、运输智能调度，帮助企业降本增效。

### 2.2 用户角色

| 角色 | 职责范围 | 典型操作 |
|------|---------|---------|
| **系统管理员** | 平台全局管理 | 账号管理、角色配置、系统参数 |
| **仓库管理员** | 仓库运营决策 | 库位规划、库存分析、盘点管理 |
| **仓库操作员** | 执行层面 | 入库登记、出库登记、拣货、盘点 |
| **调度员** | 运输指挥 | 运单分配、车辆调度、路线规划 |
| **司机/配送员** | 执行配送 | 接收任务、状态更新、轨迹记录 |
| **客户** | 下游使用方 | 下单、查询物流状态 |

### 2.3 约束条件

- **技术约束**：系统依赖 PostgreSQL + Redis + RocketMQ + MinIO，网络断开时部分功能不可用
- **数据约束**：文件存储在本地 MinIO，不上传至第三方
- **部署约束**：支持 Docker Compose 本地部署，不要求 Kubernetes
- **浏览器约束**：支持 Chrome/Firefox/Safari 最新版本

## 3. 功能需求

### 3.1 功能总览

| 模块 | 功能 | 优先级 |
|------|------|--------|
| **系统管理** | 用户管理、角色管理、菜单管理、部门管理 | 必须 |
| **仓库管理** | 库位管理、库存管理、入库管理、出库管理 | 必须 |
| **订单管理** | 订单创建、订单列表、订单详情、状态流转 | 必须 |
| **运输管理** | 运单管理、司机管理、车辆管理、配送跟踪 | 必须 |
| **数据统计** | 库存统计、订单统计、运输统计 | 应该 |
| **客户门户** | 客户注册、我要发货、物流查询 | 应该 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 页面响应时间 | ≤ 2 秒 |
| 并发用户数 | ≥ 50 |
| 数据库查询 | ≤ 500ms |
| 文件上传 | 支持最大 10MB |

### 4.2 安全性需求

| 需求 | 描述 |
|------|------|
| 密码加密 | BCrypt，不可逆 |
| JWT 有效期 | Access Token 2小时，Refresh Token 7天 |
| 权限控制 | 接口级鉴权（Spring Security） |
| 文件上传 | 白名单校验（扩展名 + MIME） |

### 4.3 可用性需求

| 指标 | 目标值 |
|------|--------|
| 系统可用率 | ≥ 99% |
| 数据备份 | 每日自动备份 |

### 4.4 日志需求

| 需求 | 描述 |
|------|------|
| 操作日志 | 关键操作（登录/增删改）记录用户+时间+操作 |
| 异常日志 | 所有异常记录堆栈信息 |
| RocketMQ 日志 | 消息发送/消费记录 |

## 5. 验收标准

### 5.1 功能验收

| 模块 | 验收标准 |
|------|---------|
| 登录 | 正确用户名密码返回 JWT；错误返回 401 |
| 库位管理 | 增删改查正常，状态显示正确 |
| 入库 | 入库单录入后库存实时增加 |
| 出库 | 出库单录入后库存实时减少，库存不足禁止出库 |
| 订单 | 状态流转符合 F23 定义的流程 |
| 运单 | 运单可分配司机，更新配送状态 |
| 权限 | 无权限用户不能访问受保护接口，返回 403 |
